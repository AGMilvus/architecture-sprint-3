# Задание 1. Анализ и планирование
### 1. Описание функциональности монолитного приложения

**Управление отоплением:**

- Пользователи могут удалённо включать/выключать отопление, установить целевую температуру
- Система поддерживает функцию удаленного включения/выключения отопления, поддержания заданной температуры.

**Мониторинг температуры:**

- Пользователи могут просматривать текущую температуру
- Система получает данные о температуре с датчиков.
- Система поддерживает веб-интерфейс пользователя для мониторинга

**Ограничения:**
- Пользователь не может самостоятельно подключить свой датчик к системе, для этого требуется выезд специалиста по подключению системы отопления.

### 2. Анализ архитектуры монолитного приложения

Архитектура приложения представляет собой монолит на Java с СУБД Postgres. Всё синхронно. Никаких асинхронных вызовов, микросервисов и реактивного взаимодействия в системе нет. Всё управление идёт от сервера к датчику. Данные о температуре также получаются через запрос от сервера к датчику.

### 3. Определение доменов и границы контекстов

- Управление отоплением. Границы контекста — данные о системе отопления, команды на включение и отключение отопления,
- Мониторинг температуры. Границы контекста — значения датчиков температуры, целевые значения температуры

### **4. Проблемы монолитного решения**

- Нет возможности масштабировать отдельные компоненты системы.
- Длительные циклы разработки и развёртывания.
- Сложность разработки монолита и как следствие высокий риск ошибок.
- Низкая отказоустойчивость, при отказе монолита, все компоненты системы недоступны.
